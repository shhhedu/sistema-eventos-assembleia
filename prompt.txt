Crie uma aplicação web full stack simples chamada "Sistema de Eventos da Assembleia de Deus".

Use as imagens anexadas pelo comando do Codex como referência visual e também como arquivos do projeto. As imagens estão na pasta `img/`. Copie ou organize essas imagens dentro de `public/assets/img/` e utilize-as como banners/logos quando fizer sentido. Se houver mais de uma imagem, escolha uma como banner padrão dos eventos e deixe as outras disponíveis para seleção ou uso futuro. Não dependa de links externos de imagem.

Regras principais:

* Frontend com HTML5, CSS3 e JavaScript puro.
* Backend com Node.js e Express.
* Banco de dados SQLite.
* Não usar React, Vue, Angular, Next.js ou TypeScript.
* Não criar sistema de membros, perfis, relatórios, gráficos ou dashboard complexo.
* O foco é simplicidade, facilidade de uso e rapidez para registrar presença em eventos da Assembleia de Deus.

Objetivo:
Permitir que visitantes visualizem eventos da igreja e confirmem presença.
Permitir que administradores criem eventos e visualizem a lista de presenças.

Fluxo do visitante:
Ao acessar o site, o visitante deve visualizar uma lista de eventos.

Cada evento deve exibir:

* Imagem/Banner
* Nome do evento
* Data
* Horário
* Local
* Resumo

Ao clicar em um evento, deve abrir uma página ou modal contendo:

* Banner
* Nome do evento
* Data
* Horário
* Local
* Descrição completa
* Área para Confirmar Presença

Para confirmar presença, o visitante deve informar:

* Nome completo
* Telefone opcional

Botão:
"Confirmar Presença"

Após confirmar:

* Salvar presença no banco de dados SQLite.
* Exibir mensagem de sucesso.
* Impedir registros duplicados com o mesmo nome para o mesmo evento.
* A validação de duplicidade deve ignorar diferença de letras maiúsculas/minúsculas e espaços extras.

Fluxo do administrador:
Criar uma área administrativa protegida por senha simples.

Credenciais padrão:
Usuário: admin
Senha: admin123

A senha pode ficar definida no backend de forma simples, pois este é um projeto acadêmico/simples.

Menu público:

* Eventos
* Administrador

Menu administrativo:

* Eventos
* Presenças

Gestão de eventos:
O administrador pode:

* Criar evento
* Editar evento
* Excluir evento
* Visualizar presenças de cada evento

Campos do evento:

* Título
* Data
* Horário
* Local
* Descrição
* Banner

Presenças:
Ao abrir um evento no painel administrativo, exibir:

* Participantes Confirmados
* Total de Confirmados: XX

Tabela de presenças:

* Nome
* Telefone
* Data da Confirmação

Banco de dados:
Tabela eventos:

* id
* titulo
* descricao
* data
* horario
* local
* banner

Tabela presencas:

* id
* eventoId
* nome
* telefone
* dataConfirmacao

Rotas esperadas no backend:

* GET /api/eventos
* GET /api/eventos/:id
* POST /api/presencas
* POST /api/admin/login
* POST /api/admin/eventos
* PUT /api/admin/eventos/:id
* DELETE /api/admin/eventos/:id
* GET /api/admin/eventos/:id/presencas

Interface:

* Visual moderno inspirado em sistemas SaaS.
* Cores principais: azul escuro, azul institucional, branco e cinza claro.
* Layout responsivo para celular e computador.
* Design limpo, com cards de eventos, botões modernos, modal bem organizado e painel administrativo simples.
* Usar as imagens anexadas como inspiração de identidade visual e, quando possível, como banners reais do sistema.

Estrutura esperada do projeto:

* package.json
* server.js
* database.js
* eventos.db gerado automaticamente
* public/

  * index.html
  * admin.html
  * css/

    * style.css
  * js/

    * app.js
    * admin.js
  * assets/

    * img/

Também crie dados iniciais de exemplo no banco caso não exista nenhum evento, usando uma imagem da pasta assets como banner padrão.

Ao terminar:

1. Explique quais arquivos foram criados.
2. Explique como instalar dependências.
3. Explique como rodar o projeto.
4. Informe a URL pública local.
5. Informe a URL da área administrativa.
6. Informe usuário e senha do admin.
7. Se algum arquivo/imagem estiver faltando, crie fallback para o sistema não quebrar.
